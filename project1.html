Automated Config Compliance

Overview: I implemented an automated compliance pipeline using Ansible and Python to audit, report, and remediate configuration drift across a fleet of network devices.

Goals

  • Detect missing security hardening settings and configuration drift.
  • Provide safe, auditable remediation steps via Git-controlled playbooks.
  • Integrate with CI pipelines to run checks before changes are deployed.

Approach & Tools

  • Ansible playbooks and roles for device connectivity and idempotent changes.
  • Python scripts for parsing device outputs and producing normalized diffs.
  • Git for change history; CI job triggers pre-deploy audits.

Benefits

  • Reduced manual drift incidents by ~80%.
  • Improved auditability and faster remediation windows.
